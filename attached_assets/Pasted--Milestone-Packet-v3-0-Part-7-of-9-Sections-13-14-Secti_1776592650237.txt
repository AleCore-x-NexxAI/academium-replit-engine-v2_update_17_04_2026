Section 14 — Acceptance Checklist
14.1 Priority 1 — Critical unfixed bugs
Student completes a session. Dashboard completion count updates within 60 seconds without hard refresh.
Completed student modal: session summary shows dashboard_summary.session_headline, never a placeholder.
Reasoning arc renders all turn dots with correct bands. Key pattern populated.

No banned superlatives in session_headline across 10 test sessions.
signal_averages computed across all turns OR both signal_averages_all and signal_averages_free_response populated.
14.2 Priority 2 — Semantic framework detection
Regression against T-002: rerun the BrewBold or equivalent Porter-shaped scenario with a student response that uses focus-strategy logic without naming Porter. Porter must be detected implicit with confidence medium or high on at least 2 of 3 applicable turns.
Regression against explicit naming: same scenario with a student who names Porter. Must detect explicit with confidence high.
Regression against unrelated reasoning: same scenario with a student whose reasoning is generic business management with no framework logic. Must detect not_evidenced.
Every FrameworkDetection output includes level, confidence, detection_method, evidence, reasoning. No nulls.
Semantic check prompt never produces the word "correct," "wrong," "should," "optimal," or any other prohibited term in its reasoning output.
Latency: semantic check adds no more than 1.5 seconds of additional turn latency when batched across tracked frameworks.
Consistency check (Section 12.3): when signal extractor returns strategic at STRONG but framework detector returns Porter as not_evidenced, the check promotes to implicit low and logs a disagreement event.
14.3 Priority 3 — Canonicalization and deduplication
Professor types "SWOT". Resolver matches to SWOT Analysis. Display confirms match.
Professor types "SWOT Analysis" on the same scenario. UI rejects with "already added" toast.
Professor types "Porter". Resolver matches to Porter's Generic Strategies OR Porter's Five Forces. UI presents a disambiguation if ambiguous.
Professor types an unknown framework name. Resolver returns canonicalId: null. Client falls back to suggest-framework-keywords. Framework is saved as custom.
Legacy scenarios with duplicate SWOT entries are deduplicated by the one-time migration. Log shows before and after.
All registry frameworks have coreConcepts, conceptualDescription, recognitionSignals in both Spanish and English.

framework_summary includes explicit_turns, implicit_turns, not_evidenced_turns, framework_name, canonicalId, provenance, detection_method_distribution for every framework in every completed session.
FrameworkDetection includes confidence, detection_method, reasoning on every turn output.
dashboard_reasoning_link renders in KpiFrameworksTab for every KPI movement.
mergeReflectionAnalytics unit test passes for all new fields.
Signal extractor returns confidence and marginal evidence alongside quality and extracted_text.
14.5 Priority 5 — Framework Inference Subsystem
Scenario with no target frameworks. FrameworkEditor surfaces up to 3 inferred frameworks with Suggested badges and inference_reason tooltips. Each has semantic fields populated.
Click Add on one. Moves to accepted list. Engine tracks it when student runs.
Never more than 3 inferred displayed.

Scenario with two or more target frameworks. No inference runs.
Session in active/completed state → FrameworkEditor read-only with lock banner.
Frameworks with accepted_by_professor: false never appear in framework_detections.
14.6 Priority 6 — Pedagogical Intent
Teaching Intent page required before case generation. Skipping returns an error.
Teaching goal only. Case generates with sensible defaults.
Teaching goal + targetFrameworks. Generated case visibly exercises them — framework names appear in caseContext or coreChallenge. At least one decision is structurally anchored.
Per-decision dimensions specified. Each decision's primaryDimension matches input.
Different teaching goals produce visibly different cases. Manually verify for 3 goal pairs.
Editing pedagogical intent is blocked once any session exists.
14.7 Priority 7 — Theory-Anchored Generation
DecisionPoint includes primaryDimension, dimensionRationale, targetFrameworkIds on every decision.
Dimension coverage check passes — at least ⌈stepCount × 0.66⌉ distinct dimensions across decisions.
Framework coverage check passes — at least one decision prompt yields semantic detection of primary targetFramework.
No two consecutive decisions share primaryDimension unless specified.
Framework field completeness — all frameworks in generated output have coreConcepts, conceptualDescription, recognitionSignals.
Professor review checkpoint appears. Regenerate-single-decision works. Publish blocked until review is complete.
14.8 Priority 8 — Decision-Design Rules
Stakeholder decision: options or prompt names at least two stakeholders with distinct interests.
Tradeoff decision: tradeoffSignature fully populated. Prompt names both cost and benefit.
Analytical decision: prompt contains at least one data point or causal claim.
Strategic decision: prompt presents at least two defensible paths.
Ethical decision: prompt frames tension between two legitimate goods.
14.9 Priority 9 — Calibration
Student applies framework. Narrator produces coherent consequence chain. No naming of framework.
Student does not apply framework. Narrator still coherent, surfaces one missed dimension, no shaming.
Signal extractor promotes primary-dimension signal from WEAK to PRESENT on marginal evidence. Never above.
Debrief question grounds in primary dimension when that signal is weak.
Dashboard summary weighted toward targetCompetencies when present.
Module Health: target frameworks top section, inferred secondary section, detection_method_distribution visible.
