TASK C — Calibration conservatism: Module Health target-first split, transferring threshold, Applied Course Theory threshold, semantic implicit floor
Context
Testing log T-003B (2026-04-20) documented the engine over-crediting vague student work. Concrete evidence: with two completed sessions on the same RegionMart scenario — one strong student, one vague-minimal — Module Health flagged Porter's Generic Strategies, SWOT Analysis, and Cost-Benefit Analysis as "Transferring" at the class level, and Applied Course Theory showed 2/2. The vague student should not pull class-level transfer upward to "transferring" on frameworks they barely touched. Root causes in the code:

routes.ts:4428 — Module Health status uses raw rate: rate >= 0.60 = transferring. A student counts toward appliedCount with any implicit detection at any confidence. Two students both getting weak-implicit SWOT = rate 1.0 = transferring.
routes.ts:4344 — Applied Course Theory counts a student if ANY turn has ANY detection with level === "explicit" || level === "implicit". No confidence floor. A single implicit-low anywhere flips the student to "applied."
frameworkDetector.ts Tier 2 semantic check — LLM returns applied: true, confidence: "medium" even on 8-word student inputs. There is no minimum-input-length floor.
Module Health output does not separate target frameworks (from pedagogicalIntent.targetFrameworks) from suggested/inferred ones. Packet §11.5 requires a target-first rendering.

This task fixes all four as a coherent calibration pass.

C3. Applied Course Theory threshold tightening
Same file routes.ts, around line 4344 (inside POST /api/scenarios/:scenarioId/class-stats).
Current:

  turnDetections?.some(

      (d.level === "explicit" || d.level === "implicit"),

Replace with:
tsconst hasApplied = fwDetections.some((turnDetections: any[]) =>
  turnDetections?.some(
    (d: any) =>
      eligibleIds.has(d.framework_id) &&
      (
        d.level === "explicit" ||
        (d.level === "implicit" && (d.confidence === "medium" || d.confidence === "high"))

  ),
);
Implicit-low, signal_pattern, consistency_promoted, not_evidenced do NOT count toward Applied Course Theory.
