# D1. Depth chart point colors (Spec 3.3)

**Problem:** Backend at `server/routes.ts:3225` correctly computes `color: "green" | "blue" | "amber"` per point. Frontend at `client/src/pages/ScenarioDashboard.tsx:502` ignores the color field and uses a single hardcoded color for all dots.

**Current code (ScenarioDashboard.tsx, line ~502):**
```tsx
<Line type="monotone" dataKey="avg" stroke="#378ADD" strokeWidth={2} dot={{ r: 4, fill: "#378ADD" }} />
```

**Fix:** Use a custom dot renderer that reads the `color` field from each point.

```tsx
const COLOR_MAP: Record<string, string> = {
  green: "#1D9E75",
  blue: "#378ADD",
  amber: "#BA7517",
};

const CustomDot = (props: any) => {
  const { cx, cy, payload } = props;
  const fillColor = COLOR_MAP[payload.color] || "#378ADD";
  return <circle cx={cx} cy={cy} r={4} fill={fillColor} stroke={fillColor} />;
};

// Then in the chart:
<Line 
  type="monotone" 
  dataKey="avg" 
  stroke="#378ADD" 
  strokeWidth={2} 
  dot={<CustomDot />}
/>
```

**Rules:**
- Green `#1D9E75` if avg ≥ 2.5 (Integrated)
- Blue `#378ADD` if avg ≥ 1.5 and < 2.5 (Engaged)
- Amber `#BA7517` if avg < 1.5 (Surface)
- The line stroke stays `#378ADD`. Only dot fill varies.

**Done:** Open a test dashboard with mixed depth turns. Verify dots are different colors based on avg value.

# D2. Debrief prep truncation (Spec 4.3)

**Problem:** Spec says "truncated (first 100 characters) with an expand button." Current code truncates at 200 and doesn't preserve sentence boundaries.

**Location:** `ScenarioDashboard.tsx:840` (inside `DebriefPrepTab`)

**Fix:** Truncate at 100 chars, preferring sentence boundaries.

```tsx
function smartTruncate(text: string, limit: number): string {
  if (text.length <= limit) return text;
  // Look for sentence end within limit+20 to avoid cutting mid-sentence
  const window = text.substring(0, limit + 20);
  const sentenceMatch = window.match(/^[\s\S]*?[.!?¿¡](?:\s|$)/);
  if (sentenceMatch && sentenceMatch[0].length <= limit + 20 && sentenceMatch[0].length >= limit / 2) {
    return sentenceMatch[0].trim();
  }
  return text.substring(0, limit) + "...";
}

// Then in the JSX:
{turn.studentInput.length > 100 && !expanded[turn.number]
  ? <>
      {smartTruncate(turn.studentInput, 100)}
      <button className="text-[10px] text-muted-foreground underline ml-1 bg-transparent border-none cursor-pointer p-0" 
              onClick={() => setExpanded(e => ({ ...e, [turn.number]: true }))}>
        {isEn ? "Show full response" : "Ver respuesta completa"}
      </button>
    </>
  : turn.studentInput}
```

**Rules:**
- Truncate at 100 chars, not 200.
- Try to end at a sentence boundary (`.`, `!`, `?`, `¿`, `¡`) when possible.
- Button text: "Show full response" (EN) / "Ver respuesta completa" (ES) — the spec's wording.

**Location:** `server/routes.ts`, inside the `POST /api/scenarios/:scenarioId/module-health` handler (around lines 3135–3175).

**Fix:** Replace the template description with an LLM call. Add a separate LLM call for the deeper description.

```typescript
import { generateChatCompletion } from "./openai";

${evidenceTexts.map((e, i) => `${i+1}. ${e}`).join("\n") || "(No evidence)"}
```

# D6. Turn annotations → AI-generated (Spec 3.3)

**Problem:** Turn annotations at `routes.ts:3238` are four hardcoded template sentences. Spec 3.3 requires "Generated by the AI layer using rds_band distribution data and classification event counts (NUDGE/BLOCK rates) for that turn."

**Location:** `server/routes.ts`, inside the `POST /api/scenarios/:scenarioId/depth-trajectory` handler (around lines 3217–3257).

**Fix:** For each turn, collect distribution data + classification event counts, then call LLM to describe what the data shows drove the depth.

```typescript
// Inside the for loop at each turn:
for (let t = 0; t < maxTurns; t++) {
  let sum = 0, count = 0;
  let integratedCount = 0, engagedCount = 0, surfaceCount = 0;
  let nudgeCount = 0, blockCount = 0, passCount = 0;
  
  for (const s of completed) {
    const state = s.session.currentState as any;
    const logs = state?.decisionEvidenceLogs || [];
    if (logs[t]) {
      const band = logs[t].rds_band;
      const val = band === "INTEGRATED" ? 3 : band === "ENGAGED" ? 2 : 1;
      sum += val;
      count++;
      if (band === "INTEGRATED") integratedCount++;
      else if (band === "ENGAGED") engagedCount++;
      else surfaceCount++;
      
      const classification = logs[t].classification;
      if (classification === "NUDGE") nudgeCount++;
      else if (classification === "BLOCK") blockCount++;
      else passCount++;
    }
  }
  
  const avg = count > 0 ? Math.round((sum / count) * 10) / 10 : 0;
  const color = avg >= 2.5 ? "green" : avg >= 1.5 ? "blue" : "amber";
  points.push({ turn: t + 1, avg, color });

  let label: string;
  if (t === 0) {
    label = avg >= 2.5 ? "Peaked" : avg >= 1.5 ? "Engaged" : "Surface";
  } else {
    const prev = points[t - 1].avg;
    if (avg >= 2.5) label = "Peaked";
    else if (avg >= 1.5) label = avg > prev ? "Engaged" : "Dropped";
    else label = "Surface";
  }

  // AI-generated annotation
  const priorAvg = t > 0 ? points[t - 1].avg : null;
  const prompt = isEn
    ? `Class reasoning depth at Turn ${t + 1}: avg ${avg} (${integratedCount} integrated, ${engagedCount} engaged, ${surfaceCount} surface out of ${count} students).
Classification events this turn: ${passCount} PASS, ${nudgeCount} NUDGE, ${blockCount} BLOCK.
${priorAvg !== null ? `Previous turn avg: ${priorAvg} (change: ${(avg - priorAvg).toFixed(1)}).` : "This is the first turn — no prior comparison."}

Write 1-2 sentences describing what the data shows drove the depth at this turn. Describe only — no recommendations, no teaching advice. No "should", "consider", "need to", "unfortunately", "fortunately". No exclamation marks. Write in English.`
    : `Profundidad de razonamiento de la clase en el Turno ${t + 1}: promedio ${avg} (${integratedCount} integrado, ${engagedCount} engaged, ${surfaceCount} superficial de ${count} estudiantes).
Eventos de clasificación este turno: ${passCount} PASS, ${nudgeCount} NUDGE, ${blockCount} BLOCK.
${priorAvg !== null ? `Promedio del turno anterior: ${priorAvg} (cambio: ${(avg - priorAvg).toFixed(1)}).` : "Este es el primer turno — no hay comparación previa."}

Escribe 1-2 oraciones describiendo lo que los datos muestran que impulsó la profundidad en este turno. Solo descripción — sin recomendaciones, sin consejos pedagógicos. Sin "debería", "considera", "necesita", "lamentablemente", "afortunadamente". Sin signos de exclamación. Escribe en español.`;

  let description: string;
  try {
    description = (await generateChatCompletion(
      [{ role: "user", content: prompt }],
      { maxTokens: 150, model: "gpt-4o-mini", agentName: "depthTurnAnnotation" }
    )).trim().replace(/^["']|["']$/g, "");
  } catch {
    description = isEn
      ? `Average depth ${avg} at Turn ${t + 1} with ${nudgeCount} nudge and ${blockCount} block events.`
      : `Profundidad promedio ${avg} en Turno ${t + 1} con ${nudgeCount} eventos de nudge y ${blockCount} de block.`;
  }

  annotations.push({ turn: t + 1, label, description });
}
```

**Rules (enforce post-generation):**
- Max 2 sentences.
- Describes what the data shows.
- Never prescriptive.
- Same prohibited language list as D5.
- Uses scenario language.

**Done:** Each turn has an AI-generated annotation referencing the actual distribution + classification data.

# D8. Class debrief opener → AI-generated (Spec 3.4)

**Problem:** Class debrief opener at `routes.ts:3176` is hardcoded. Spec 3.4 requires:
1. Identify the competency with the lowest class demonstration rate
2. Cross-reference with the framework with the lowest module health status
3. Generate a question that connects both AND references a specific decision point (turn where the gap was most pronounced)

**Location:** `server/routes.ts`, inside `POST /api/scenarios/:scenarioId/module-health` (around lines 3176–3179).

**Fix:** Compute the lowest competency, lowest framework, and the turn where each was most absent, then generate the opener via LLM.

```typescript
// After computing frameworkResults (from D5), and before the final result:

// Step 1: Compute per-competency rates (same as class-patterns endpoint)
const compMap: Record<string, { name: string; nameEs: string; rate: number; worstTurn: number | null }> = {
  C1: { name: "Analytical reasoning", nameEs: "Razonamiento analítico", rate: 0, worstTurn: null },
  C2: { name: "Strategic decision-making", nameEs: "Toma de decisiones estratégicas", rate: 0, worstTurn: null },
  C3: { name: "Stakeholder consideration", nameEs: "Consideración de stakeholders", rate: 0, worstTurn: null },
  C4: { name: "Ethical reasoning", nameEs: "Razonamiento ético", rate: 0, worstTurn: null },
  C5: { name: "Tradeoff awareness", nameEs: "Conciencia de tradeoffs", rate: 0, worstTurn: null },
};

// Compute rates and find the turn where each competency was most absent
for (const [key, info] of Object.entries(compMap)) {
  let count = 0, total = 0;
  const turnAbsence: Record<number, { absent: number; total: number }> = {};
  for (const s of completed) {
    const state = s.session.currentState as any;
    const logs = state?.decisionEvidenceLogs || [];
    logs.forEach((log: any, i: number) => {
      total++;
      const turnNum = i + 1;
      if (!turnAbsence[turnNum]) turnAbsence[turnNum] = { absent: 0, total: 0 };
      turnAbsence[turnNum].total++;
      const ev = log.competency_evidence?.[key];
      if (ev === "demonstrated" || ev === "emerging") count++;
      else turnAbsence[turnNum].absent++;
    });
  }
  info.rate = total > 0 ? count / total : 0;
  
  // Find turn with highest absence rate
  let maxAbsRate = 0;
  for (const [turnStr, data] of Object.entries(turnAbsence)) {
    const absRate = data.total > 0 ? data.absent / data.total : 0;
    if (absRate > maxAbsRate) {
      maxAbsRate = absRate;
      info.worstTurn = Number(turnStr);
    }
  }
}

- References Turn ${targetTurn} specifically
- Connects the framework and the competency
- Is a genuine open question (ends with "?")
- Asks students to name the framework they used AND consider a dimension they missed
- Must not imply any student was wrong
- No "should have", "what was the right answer", "correct", "incorrect"
- No exclamation marks
- Write in English`
    : `Vas a generar una pregunta de apertura de debrief para el profesor.
Competencia más baja: "${compName}" (demostrada en solo ${Math.round(lowestComp[1].rate * 100)}% de las evaluaciones)
Marco con menor transferencia: "${lowestFw.name}" (estado: ${lowestFw.status})
```

# D9. Framework detection evidence → richer generation (Spec 4.5 + 5.1)

**Problem:** `frameworkDetector.ts` currently generates templated evidence strings like `'The response directly references "X", a key term from the Y framework.'` Spec 5.1 and 4.5 require the evidence to describe "what in the student's input produced this classification" and "reference the student's specific reasoning where applicable" — max 1 sentence.

The spec doesn't strictly require LLM generation here (the signal-pattern detection is rule-based), but the evidence sentence needs to be more grounded in the student's actual words, not just "matched a keyword."

**Location:** `server/agents/frameworkDetector.ts`

**Fix:** Make the evidence sentence richer by (a) quoting a relevant fragment from the student's input, (b) naming the specific concept/keyword matched, and (c) explaining the connection to the framework.

```typescript
// Replace the evidence generation with:

// EXPLICIT case:
if (nameMatch || keywordMatch) {
  const matchedTerm = nameMatch ? fw.name : fw.domainKeywords.find((kw) =>
    new RegExp(`\\b${escapeRegex(kw)}\\b`, "i").test(studentInput)
  ) || fw.name;
  
  // Extract a short fragment (up to 60 chars) containing the matched term
  const termIdx = studentInput.toLowerCase().indexOf(matchedTerm.toLowerCase());
  const start = Math.max(0, termIdx - 20);
  const end = Math.min(studentInput.length, termIdx + matchedTerm.length + 20);
  const fragment = studentInput.substring(start, end).trim().replace(/^\W+|\W+$/g, "");
  
  return {
    framework_id: fw.id,
    framework_name: fw.name,
    level: "explicit" as const,
    evidence: language === "en"
      ? `Student referenced "${matchedTerm}" directly in the context "…${fragment}…", a core term of the ${fw.name} framework.`
      : `El estudiante mencionó "${matchedTerm}" directamente en el contexto "…${fragment}…", un término central del marco ${fw.name}.`,
  };
}

// IMPLICIT case:
if (allSignalsMet && (hasAdditionalKeyword || fw.domainKeywords.some(...))) {
  // Find which domain concept the student engaged with

                    additionalKws.find((kw) => inputLower.includes(kw.toLowerCase()));
  
  const sigLabels = fw.signalPattern.requiredSignals.map(s => {
    const labels: Record<string, { en: string; es: string }> = {
      intent: { en: "strategic decision-making", es: "toma de decisiones estratégicas" },
      justification: { en: "analytical reasoning", es: "razonamiento analítico" },

      ethicalAwareness: { en: "ethical reasoning", es: "razonamiento ético" },
    };
    return labels[s]?.[language === "en" ? "en" : "es"] || s;
  }).join(language === "en" ? " and " : " y ");
  
  return {
    framework_id: fw.id,
    framework_name: fw.name,
    level: "implicit" as const,
    evidence: language === "en"
      ? `Response applied ${fw.name} logic through ${sigLabels}${matchedKw ? ` around "${matchedKw}"` : ""} without naming the framework.`
      : `La respuesta aplicó la lógica de ${fw.name} mediante ${sigLabels}${matchedKw ? ` alrededor de "${matchedKw}"` : ""} sin nombrar el marco.`,

// NOT_EVIDENCED case:
return {
  framework_id: fw.id,

  level: "not_evidenced" as const,
```

**Rules:**
- Max 1 sentence.
- Quote a short fragment from the student's input when explicit match (using the `"…fragment…"` format).
